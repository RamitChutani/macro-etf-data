# Session Worklog - 2026-03-16

## Goal
Debug and verify the "FX Jan 1st CAGR %" column in the main dashboard screener table, check for errors, and validate the output makes sense.

## Analysis

### Initial Investigation
- Reviewed `src/build_excel_dashboard_mvp.py` to trace FX Jan 1st CAGR calculation flow
- Found the calculation at lines 696-710 (ticker rows) and 826-843 (GDP-only rows)
- Identified the data source: `compute_annual_jan1_fx_returns()` from `src/build_combined_etf_weo.py`

### Critical Finding: Wrong Currency Used
**The FX Jan 1st CAGR column was calculating the wrong metric:**

| Column | Implementation | Uses |
|--------|---------------|------|
| **FX CAGR %** (Column F) | `fx_level_cagr(gdp_current_lcu_map, gdp_current_usd_map, country_code, ...)` | ✅ Country LCU vs USD (from IMF WEO GDP levels) |
| **FX Jan 1st CAGR %** (Column G) | `quote_fx_jan1_map.get((ccy_norm, y))` | ❌ ETF quote currency vs USD (from Yahoo FX) |

**Example mismatches:**
- Australia (LCU=AUD): Primary ETF SAUS.L trades in GBp → showed GBP/USD instead of AUD/USD
- Brazil (LCU=BRL): Primary ETF XMBR.L trades in GBp → showed GBP/USD instead of BRL/USD
- Japan (LCU=JPY): Primary ETF LCJP.L trades in GBP → showed GBP/USD instead of JPY/USD

### Additional Issues Found
1. **Syntax Error in `src/build_combined_etf_weo.py`** (lines 445-455):
   - Missing closing bracket `]` for column list
   - Duplicate `return merged` statements (dead code)

## Fix Implementation

### Files Modified

1. **`src/etf_mapping.py`**
   - Added `COUNTRY_TO_LCU` dictionary mapping country names to local currencies (AUD, BRL, EUR, JPY, GBP, etc.)

2. **`src/build_combined_etf_weo.py`**
   - Fixed syntax error (removed duplicate returns, closed bracket properly)
   - Added import for `COUNTRY_TO_LCU`
   - Compute `country_lcu_fx_jan1_map` for all country LCUs
   - Added `country_lcu_vs_usd_jan1_pct` column to output CSV

3. **`src/build_excel_dashboard_mvp.py`**
   - Added import for `COUNTRY_TO_LCU` and `compute_annual_jan1_fx_returns`
   - Compute `country_lcu_fx_jan1_map` at dashboard build time
   - Updated ticker row calculation (lines 695-716) to use `COUNTRY_TO_LCU.get(country_name)` instead of ETF quote currency
   - Updated GDP-only row calculation (lines 826-843) to use country LCU instead of first ticker's currency
